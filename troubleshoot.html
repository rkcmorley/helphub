Upgrading certificates in User Acceptance Testing (UAT)

Before we upgrade the certificates live into our products and systems, we will make the changes within our UAT environment. This will give us, and our clients who access UAT, time to ensure that everything works, without affecting the live product.

Certificates for Experian websites in UAT

Sunday 29 November 2015 ( 4.30 - 9.30pm )

Certificates for web services that make up the Experian written security system (known as WASP*) in UAT

Wednesday 2 December 2015 ( 6pm - 0.15 am )

Certificates for the rest of our web services in UAT

Sunday 6 December 2015 ( 4.30 - 9.30pm )

Issues within UAT

If you have an integrated system and have access to our User Acceptance Testing (UAT) environment: then you should test compatibility with the SHA-256 Certificates between Sunday the 29 November & 6 December.

If you experience any connectivity issues then we would advise that you investigate for the following compatibility issues.

1. You have an internal networking system or security devices in place (such as a Proxy Gateway, Network Concentrator or Load Balancer).

a. If this is the case you may need to download and install a new SHA-256 certificate key chain.

b. You can download the new SHA-256 Certificates from our Certificate Authority, Entrust, from the links below. These links also provide installation instructions.

2. You access our systems via a web browser and you (or your teams) are not using a supported browser:

a. You may experience problems if you (or your teams) are not using a supported browser (typically lower than IE 6, Service Pack 3). You can find out more about compatible systems here: https://www.digicert.com/sha-2-compatibility.htm

b. If this is the case you may need to install a new SHA-256 certificate key chain on your browser. If you find this resolves issues you will need to ensure this is updated across your IT internal environment.

c. If you are accessing the internet from a browser lower than IE 6 (Service Pack 3) you may need to update your browser. But few clients are likely to have this problem as web browser analytics show less than 0.2% of the population is on IE 6.

Please note: you can do this any time (i.e. you don't need to wait until we upgrade our certificates).
